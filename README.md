# ConfigConverter — транслятор учебного конфигурационного языка в YAML (Вариант №23)

## Назначение
`ConfigConverter` — консольная утилита на Java, которая:
- читает входной текст на учебном конфигурационном языке из **STDIN**;
- выявляет синтаксические ошибки и пишет сообщения в **stderr**;
- преобразует данные в **YAML** и сохраняет в файл, путь к которому задаётся ключом командной строки.

## Возможности языка (поддержано)
- **Однострочные комментарии**:  
  `REM Это комментарий до конца строки`
- **Многострочные комментарии**:

(comment
Это многострочный
комментарий
)

- **Числа**: `[+-]?\d+\.\d+` (например `1.0`, `-0.25`)
- **Строки**: `@"Это строка"`
- **Массивы**: `[ значение; значение; ... ]`
- **Словари**:

{
имя : значение,
имя : значение
}

Имена ключей: `[a-z]+`
- **Константы**: `имя := значение`
- **Константные выражения (ОПН)**: `| ... |`  
Операции/функции: `+`, `-`, `*`, `concat()`, `mod()`

> Результатом трансляции считается **последнее** встреченное выражение, которое **не является объявлением константы**.

## Требования
- Java 17+ (подойдёт и более новая версия)

## Сборка
### Вариант 1 — обычная компиляция
bash
javac ConfigConverter.java ConfigConverterTest.java

Запуск утилиты

Формат:

java ConfigConverter -o output.yaml < input.txt

Пример:

java ConfigConverter -o result.yaml < examples/shop.conf

Запуск тестов

Тесты используют assert, поэтому важно включить проверки ключом -ea:

java -ea ConfigConverterTest

Если какой-то тест падает — процесс завершится с кодом 1.
Примеры входных конфигураций
1) Интернет-магазин (скидки и заказ)

Вход:

REM Константы
price := 199.99
qty := 3.0

{
  item : @"phone",
  total : |price qty *|,
  tags : [ @"sale"; @"new" ]
}

Выход (пример YAML):

item: "phone"
total: 599.97
tags:
- "sale"
- "new"

2) Учебный журнал (массив объектов)

Вход:

{
  group : @"10A",
  students : [
    { name : @"Ivan", score : 4.0 };
    { name : @"Anna", score : 5.0 }
  ]
}

3) Техника/IoT (конкатенация списков)

Вход:

base := [ @"temp"; @"humidity" ]
{
  sensors : |base [ @"pressure" ] concat()|
}

Типы данных в результате

    числа -> Double

    строки -> String

    массивы -> List<Object>

    словари -> Map<String, Object>

    константы -> сохраняются в таблице констант и подставляются при использовании

Структура проекта

    ConfigConverter.java — CLI + парсер + YAML-генератор

    ConfigConverterTest.java — набор тестов (без сторонних библиотек)

    report.md — отчёт/описание решения (если приложен)

Типичные ошибки и диагностика

При ошибках синтаксиса утилита завершает работу с кодом 1 и выводит сообщение вида:

<описание ошибки> (строка N, позиция M)

Лицензия

Учебный проект (по заданию). Используйте свободно для учебных целей.
